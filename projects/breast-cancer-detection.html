Breast Cancer Detection Using Deep Learning

AI-Powered Histopathology Image Analysis for Early Cancer Detection

Problem Statement

Breast cancer is one of the leading causes of death among women worldwide. Early and accurate detection is critical for improving treatment outcomes and survival rates. However, the current diagnostic process relies heavily on manual examination of histopathology images by pathologists, which is time-consuming, prone to human error, and limited by the shortage of specialists, especially in low-resource settings. These challenges often lead to delays and inconsistencies in diagnosis. Therefore, there is a need for an accurate, efficient, and accessible system that can assist pathologists in detecting breast cancer from histopathology images using deep learning techniques.

How It Works

This deep learning system analyzes histopathology images to classify tissue as cancerous or non-cancerous, enabling rapid screening and early detection to support clinical decision-making.

Development Process

1. Environment Setup & GPU Configuration

Set up Google Colab with GPU acceleration for efficient deep learning training. Verified TensorFlow installation and GPU availability for faster computations.

2. Dataset Acquisition

Downloaded the Breast Histopathology Images dataset (277,524 images) from Kaggle. Dataset contains 50x50 pixel patches, labeled as cancerous (1) or non-cancerous (0).

3. Data Preprocessing & Organization

Organized dataset into training/validation splits (80/20 ratio). Implemented random shuffling and limited to 6,000 images per class to optimize for free-tier GPU constraints while maintaining class balance.

4. Data Pipeline Optimization

Created efficient data loading using TensorFlow's image_dataset_from_directory. Resized images to 224x224 pixels (ResNet50 input size) and implemented caching, shuffling, and prefetching for maximum GPU utilization.

5. Model Architecture Design

Built transfer learning model using ResNet50 pre-trained on ImageNet. Froze base layers and added custom classification layers: Global Average Pooling, Dense(128, ReLU), Dropout(0.5), and final Dense(2, softmax) for binary classification.

6. Model Training & Optimization

Compiled with Adam optimizer and categorical cross-entropy loss. Implemented early stopping (patience=3) and model checkpointing to save best weights based on validation accuracy. Trained for up to 10 epochs.

7. Model Evaluation & Validation

Evaluated on validation set using validation accuracy and loss.

8. Web Application Development

Built Streamlit web application for easy deployment. Created intuitive interface where users can upload histopathology images and receive instant predictions with confidence scores and visual feedback.
